### `chatbot`
완전 로컬 환경에서 동작하는 RAG 챗봇 계열 프로젝트. 문서 업로드, 벡터 기반 검색, 스트리밍 응답 같은 흐름을 의도한 것으로 보인다.

### `data_generation`
OpenAI API를 활용해 반도체 도메인 데이터셋을 생성하고, 온톨로지/구조화 작업을 수행한 프로젝트. 단순 생성이 아니라 품질 가드레일까지 신경 쓴 흔적이 있다.

### `deeptech`
PECVD lot log, 두께 데이터, 공정 로그 같은 제조 데이터를 분석하는 작업 폴더. 그중 `iste_dataset_viewer`는 Next.js 기반 분석/시각화 툴 성격이 강하다.

### `ir`

## 내가 어필할 수 있는 것

### 가장 강한 포지셔닝
나는 반도체/제조 도메인을 이해하면서, 그 문제를 AI 시스템과 실무형 소프트웨어로 연결할 수 있는 개발자라고 어필할 수 있다.

보통은 아래 둘 중 하나에 치우친 경우가 많다.
- AI/소프트웨어는 할 수 있지만 반도체 제조 도메인을 잘 모른다.
- 반도체 제조 도메인은 알지만 직접 시스템으로 구현하지 못한다.

지금까지의 작업 이력은 이 둘을 실제로 연결해 본 경험으로 읽힌다.

### 핵심 어필 포인트
- 반도체 제조/설비 데이터 이해
  - WAT, Bin Map, Yield, Datalog, lot log, 공정/설비 데이터 등 현업형 데이터를 다뤘다.
- AI를 실제 문제에 붙인 경험
  - 이상감지, 예지보전, RAG, OCR, 멀티에이전트, 설명형 AI까지 구현 축이 이어진다.
- 데이터부터 UI까지 end-to-end로 연결하는 능력
  - 수집, 정제, 데이터셋, DB, 파이프라인, 백엔드, 프론트, 시각화, 문서화까지 모두 손댔다.
- 문서 기반 지식체계 구축 능력
  - OCR, 문서 파싱, ontology, 관계 추출, 검색/질의응답 파이프라인을 직접 구성했다.
- 실험에서 끝나지 않는 실행력
  - 요구사항정의서, API 명세서, DB 설계서, 개발 가이드, 배포 가이드, 발표자료까지 만들었다.
- 현업 적용 감각
  - 단순 모델 정확도보다 실제 업무 흐름과 운영 구조로 이어지는 형태를 반복해서 만들었다.

### 한 줄로 어필하면
- 반도체 제조 데이터를 이해하고, 이를 AI 시스템과 실무형 소프트웨어로 연결하는 엔지니어
- OCR, RAG, 이상감지, 제조 데이터 분석을 end-to-end로 구현하는 AI/소프트웨어 엔지니어
- 반도체·제조 도메인 문제를 데이터 파이프라인, AI, 웹 시스템으로 제품화하는 개발자

## 이력서/포트폴리오에서 밀어야 할 메시지

### 1. 반도체/제조 + AI를 같이 할 수 있다
가장 차별점이 큰 지점이다. 도메인 이해와 구현 능력이 같이 있다는 점을 전면에 두는 게 좋다.

### 2. PoC가 아니라 시스템 단위로 만든다
단발성 노트북 실험이 아니라, 수집-저장-분석-서비스-문서화로 이어지는 구조를 만든 경험이 있다.

### 3. 문서와 데이터를 지식 시스템으로 바꾸는 능력이 있다
OCR, 문서 파싱, ontology, RAG를 연결해서 검색 가능한 구조와 응답 시스템으로 바꿨다.

### 4. 현업형 분석 도구를 직접 만들 수 있다
수율, WAT, Bin Map, 로그 데이터처럼 제조 현장에서 실제로 쓰이는 데이터를 다루는 툴을 만들었다.

### 5. 고객/조직 커뮤니케이션까지 감당한다
기술 문서, 요구사항 정의, 설계서, 발표자료까지 작성했다는 점은 단순 개발자보다 한 단계 넓은 역할 수행 경험으로 볼 수 있다.
